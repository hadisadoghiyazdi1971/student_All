# project

This folder contains the modeling and reporting stage of the health digital-twin pipeline.
It consumes prepared data from `../DataPreparation/merged_health_data.csv`, builds a composite health index, trains prediction models, generates recommendations, and writes final reports.

## Main Artifacts in This Folder

- `health_digital_twin.ipynb` (core modeling + reporting notebook)
- `requirements.txt` (Python dependencies)
- `final_health_report.txt` (final textual report generated by notebook)
- `Final Model Comparison.png` (model comparison visual)
- `Health.pdf` (exported report document)
- `.env` (optional environment variables, usually API keys)
- `latex_report/` (report presentation files)

## Notebook Explained: `health_digital_twin.ipynb`

This notebook is the core implementation of your digital-twin logic.

### Stage 1: Imports and setup

Libraries loaded:

- Data and math: `pandas`, `numpy`, `math`, `json`
- Visualization: `matplotlib`
- ML: `LinearRegression`, `RandomForestRegressor`, `TimeSeriesSplit`, `GridSearchCV`, metrics
- Environment/API: `dotenv`, `google.generativeai`
- Additional model package: `xgboost` (imported, not central in current notebook path)

### Stage 2: Load prepared dataset

- Reads `../DataPreparation/merged_health_data.csv`
- Parses `date` and sorts chronologically

### Stage 3: Build enhanced health index (`HI_v2`)

The notebook builds 5 sub-scores then combines them into a single daily index.

Sub-scores:

- `activity_score` from `daily_step_count`
- `cardio_score` from average/max/min heart-rate signals
- `resp_score` from SpO2 average/max/min
- `posture_score` from shoulder asymmetry + slouch angle
- `vocal_score` from jitter/shimmer/HNR/F1/F2

Final weighted index:

- `HI_v2 = 0.35*activity + 0.25*cardio + 0.20*resp + 0.10*posture + 0.10*vocal`

### Stage 4: Feature engineering and supervised target

- Creates next-day target: `HI_v2_next = HI_v2.shift(-1)`
- Drops `NaN` from shift
- Uses 5 sub-scores as model inputs
- Standardizes features using `StandardScaler`
- Splits data with chronological 80/20 split

### Stage 5: Train/evaluate models

Model 1:

- Linear Regression with `TimeSeriesSplit` cross-validation
- Tracks MAE and RMSE across folds

Model 2:

- Random Forest with `GridSearchCV` + `TimeSeriesSplit`
- Hyperparameters tuned: estimators, depth, split/leaf size, max_features
- Evaluated on test set MAE/RMSE

### Stage 6: Recommendation engine

- Rule-based function `generate_recommendation_v2(row)`
- Uses metric thresholds for SpO2, heart rate, posture, voice jitter, and steps
- Collects possible actions, assigns priority, returns highest-priority recommendation
- Prints recent-day status with current HI, predicted HI, and recommendation

### Stage 7: Visualization

- Plots real vs predicted HI values (Linear Regression and Random Forest)
- Saves model comparison visuals (for example `Final Model Comparison.png`)

### Stage 8: LLM narrative analysis

- Builds `summary_text` with model performance and HI statistics
- Optionally calls Gemini (`gemini-2.5-flash`) using `GOOGLE_API_KEY`
- Produces one-paragraph scientific interpretation text

### Stage 9: Final report export

- Writes combined summary + Gemini analysis to `final_health_report.txt`

## How This Folder Connects to DataPreparation

Input dependency:

- This notebook expects the merged file produced upstream in `../DataPreparation`.

End-to-end order:

1. Run preprocessing in `../DataPreparation` to refresh `merged_health_data.csv`.
2. Run `health_digital_twin.ipynb` in this folder.
3. Review generated plot/report files and feedback outputs.

## Quick Start

1. Create and activate your Python environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Ensure upstream file exists:

- `../DataPreparation/merged_health_data.csv`

4. Set environment variable for Gemini (optional):

```bash
# PowerShell example
$env:GOOGLE_API_KEY="your_key_here"
```

5. Run `health_digital_twin.ipynb` top to bottom.

## Security Notes

- Keep API keys in `.env` or environment variables, not in notebook cells.
- Some upstream notebooks currently hard-code keys; rotate/remove those keys before publication.

## Known Practical Constraints

- The rule-based recommendation thresholds are currently static constants; results are sensitive to those values.
- Time-series size appears limited; model metrics may vary substantially with new data.
